# Project Overview — AF Nexus Community

## System Name
AF Nexus — Community Module

---

## What Is This?

AF Nexus Community is a neighborhood/community social platform built on top of the existing **AFHome (Apsara Home)** e-commerce ecosystem.

It is NOT a standalone app. It is an **extension layer** — users log in once through AFHome and the community features become available without a second login or separate account.

---

## Platform Reference (UI Model)

The community UI is modeled after **Nextdoor / Neighborly** style:
- Neighborhood-based feed
- Location-aware posts and events
- Community groups
- Safety alerts
- Local marketplace listings
- Neighbor profiles and activity

---

## Core Principle

> Single Authentication — AFHome credentials are the identity system.

- No separate signup or login for the community module
- The AFHome session token is validated by the community backend
- User profile data (name, avatar, tier) is pulled from AFHome

| Layer | Technology |
|---|---|
| Framework | Laravel (PHP) |
| Database | MySQL / PostgreSQL |

---

## Integration Architecture

```
AFHome User (Logged In)
        ↓
AFHome Auth Token (JWT / Cookie)
        ↓
Community Backend (Hono.js)
  → Validates token against AFHome API
  → Fetches user identity (id, name, avatar, tier)
        ↓
Community Features (Feed, Posts, Events, Groups)
```
